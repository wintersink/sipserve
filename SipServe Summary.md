# SipServe Summary

## Project Overview

SipServe is an autonomous beverage delivery robot built on a Raspberry Pi 5 with differential-drive tank treads. It uses an OAK-D-Lite stereo camera for AprilTag-based navigation, 5 TK50 ultrasonic sensors for obstacle detection, a DF2301QG offline voice recognition module for ordering, a 20×4 I2C LCD for status, and a Yahboom 4-channel Motor Encoder Driver for locomotion. All peripherals (except the camera) share I2C bus 1; the ultrasonics and voice sensor sit behind a TCA9548A mux.

SSH: user `testbot`, hostname `sipserve-johann`.

## Running the Code

```bash
# Activate the virtual environment (uses system site-packages)
source venv4sipserve/bin/activate

# Run the full delivery state machine
python main.py

# Smoke tests (require hardware connected)
python smoketests/test_i2c.py                 # Detect Yahboom driver at 0x26
python smoketests/IIC.py                      # Yahboom motor driver reference script
python smoketests/test_control_motor.py       # hardware.motor.Motor wrapper sanity check
python smoketests/test_motor_calibration.py   # Measure mm/sec and deg/sec
python smoketests/test_ultrasonic_array.py    # Poll all 5 mux-selected ultrasonics
python smoketests/test_obstacle_avoidance.py  # control/avoid reactive demo
python -m control.avoid                       # Same, via module entry
python sensors/voice.py                       # DF2301QG command ID tail
python sensors/apriltag.py                    # Tag detection + stereo distance demo
python smoketests/calibrate_tags.py           # Rewrite config.py TAG_IDS mapping
python smoketests/test_force_sensor.py        # Drink-tray FSR load classifier
python smoketests/test_force_sensor.py --calibrate  # Print raw ADC counts for threshold tuning
```

No build step — pure Python.

## Architecture

```
main.py                     ← Top-level SipServe FSM (see states.md)
config.py                   ← TAG_IDS: destination-name → AprilTag ID
                              (auto-generated by smoketests/calibrate_tags.py)
states.md                   ← Authoritative state-machine reference

hardware/motor.py           ← Motor class: Yahboom I2C driver with forward/reverse/
                              turn + time-based move_distance / rotate. Encoders
                              wired but unreliable — navigation uses dead
                              reckoning plus vision.
hardware/mux.py             ← TCA9548A (0x70) channel selector; SENSOR_CHANNELS map
hardware/lcd.py             ← SipServeLCD: PCF8574 20×4 status display (RPLCD)

sensors/apriltag.py         ← Standalone AprilTag 36h11 detector + stereo depth
                              demo. Exports constants reused by main.py.
sensors/oak_pipeline.py     ← Legacy/expanded OAK pipeline (not used by main.py)
sensors/ultrasonic.py       ← TK50 (0x57) driver + UltrasonicArray helper
sensors/voice.py            ← DF2301QG (0x64, mux ch 5) voice command polling

control/avoid.py            ← Reactive obstacle-avoidance standalone demo. Not
                              shared with main.py. main.py's FSM only uses
                              front ultrasonics to HALT in LOCKED_ON; the roam
                              phase in SEARCHING/LOST implements its own avoidance.

smoketests/                 ← Bench scripts. IIC.py is the Yahboom reference;
                              test_delivery_run.py is a prior monolithic delivery
                              script that predates the main.py FSM refactor.
```

### State Machine

The authoritative reference is `states.md`. Summary:

- **INITIALIZING** — scan-rotate for the Home tag, then pulse-and-pause center it.
- **AWAITING_ORDERS** — stationary; voice commands build the delivery queue.
- **SEARCHING** — alternate a roam phase (reactive obstacle avoidance) and a scan phase (burst-rotate up to 960°) until the target tag is detected.
- **LOCKED_ON** — center the tag (bang-bang tolerance 30 px), then drive forward with bang-bang steering. Arrival when stereo depth < 12" or tag visible and front ultrasonic < 12".
- **LOST** — target disappeared mid-approach; re-enter roam/scan cycle.
- **DELIVERED** — pause 10 s at destination; pop queue. If empty → RETURNING.
- **RETURNING** — same approach logic as LOCKED_ON targeting the Home tag.

### main.py Runtime Structure

- `SipServe` owns the FSM, motor, LCD, and three background workers:
  - `UltrasonicPoller` — cycles through all 5 sensors via the mux; publishes `snapshot()`, `front_min_in()`, `sides_min_in()`.
  - `VoicePoller` — drains DF2301QG command IDs into a deque.
  - `TagWatcher` — runs the depthai pipeline, maintains `{tag_id: {px, dist_mm, last_seen}}` + detected/lost events.
- All I2C access (mux + muxed sensors) goes through a shared `threading.Lock`.
- Voice IDs: 2 = wake, 5–8 = queue Table 1–4, 9 = clear, 10 = start delivery.

### Motor Conventions (hardware/motor.py)

- I2C address `0x26`, PWM control register `0x07`, 8 bytes (4× big-endian signed int16).
- **M1 = right tread, M3 = left tread; M2/M4 unused.**
- **Negative PWM drives forward** (wiring convention — `forward()` sends `-speed`).
- PWM range: −7200…7200. Defaults: `FULL_SPEED=720`, `SLOW_SPEED=360`, `TURN_SPEED=360`.
- Time-based movement calibrated empirically: `MM_PER_SEC_AT_FULL=288`, `DEG_PER_SEC_AT_TURN=84`.
- Motors are 520-type with 55 mm wheels, 45:1 gearbox, 11 PPR encoders, 154 mm track width.

Note: `control/avoid.py` uses a *different* motor convention (M1 = left, positive = forward, PWM up to 1000). Standalone demo — do not cross-wire with main.py.

## Key Hardware Constants (I2C bus 1)

| Device                          | Address | Notes                              |
|---------------------------------|---------|------------------------------------|
| Yahboom 4-ch Motor Driver       | `0x26`  | PWM register `0x07`                |
| TCA9548A mux                    | `0x70`  | Ultrasonics on ch 0,1,2,3,7; voice on ch 5 |
| TK50 ultrasonic (×5)            | `0x57`  | Shared address, selected via mux   |
| DF2301QG voice recognition      | `0x64`  | Behind mux channel 5               |
| DFRobot LCD (PCF8574 backpack)  | `0x20`  | 20×4 characters                    |
| ADS1115 ADC (drink-tray FSR)    | `0x48`  | Not muxed; FSR on ch 0, gain 1. Thresholds in `smoketests/test_force_sensor.py` are still placeholders — re-run with `--calibrate`. Smoke-test only; not wired into `main.py`. |
| I2C pins                        | —       | SDA = Pin 3, SCL = Pin 5           |

Ultrasonic layout (mux channel → name): 0 left, 1 front_left, 2 front, 3 front_right, 7 right. Obstacle threshold = 305 mm (12 in).

AprilTag family: 36h11. Detection resolution 640×480 (RGB on CAM_A); stereo depth from 640×400 mono on CAM_B/CAM_C with extended disparity.

## Dependencies (requirements.txt)

- `smbus2` — I2C
- `pigpio` — legacy GPIO/PWM; daemon required if used (`sudo pigpiod`). The current Motor class does **not** use pigpio — it drives the Yahboom board over I2C.
- `depthai` — OAK-D-Lite pipeline (v3 API)
- `RPLCD` — I2C character LCD

Also needs (not in requirements.txt):
- `opencv-python-headless` — used by some smoketests for display
- `Adafruit_ADS1x15` — only for `smoketests/test_force_sensor.py`. On Pi 5 the constructor must be called as `ADS1115(busnum=1)`; auto-detection raises "Could not determine default I2C bus".
